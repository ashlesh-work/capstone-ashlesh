# Qué son las WCAG

Las Directrices de Accesibilidad para el Contenido Web (WCAG) son el estándar técnico internacional para la accesibilidad digital, publicado por el Consorcio de la World Wide Web (W3C). Las WCAG organizan los requisitos en "criterios de conformidad" evaluables en tres niveles de conformidad: A, AA y AAA.

# Los cuatro principios POUR

Las WCAG agrupan sus criterios bajo cuatro principios. El contenido debe ser Percibible (los usuarios pueden percibir la información), Operable (los usuarios pueden operar la interfaz), Comprensible (la información y la operación son claras) y Robusto (el contenido funciona de manera confiable con las tecnologías asistivas actuales y futuras).

# Versiones

Las WCAG 2.0 (2008) están incorporadas en la Sección 508. Las WCAG 2.1 (2018) agregaron criterios para dispositivos móviles, baja visión y aspectos cognitivos, y son adoptadas por la norma del Título II de la ADA. Las WCAG 2.2 (2023) se basan en la versión 2.1, son compatibles con ella e introducen nuevos criterios para el foco, el tamaño del objetivo y la autenticación. El W3C recomienda utilizar la versión más reciente.

# Objetivo recomendado de ingeniería

Para un desarrollo nuevo, cumplir con las WCAG 2.2 Nivel AA es el objetivo de ingeniería más preparado para el futuro, ya que satisface las obligaciones de la versión 2.1 AA y reduce la necesidad de remediación posterior.
