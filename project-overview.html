Problem framing

Staff need fast, trustworthy, cited answers

Scenario: Customer Support — AI Support Resolution Agent, applied to digital-accessibility (ADA / Section 508) support, with the safety requirements of that scenario adopted.

P

Priya, Digital Compliance Coordinator

At a mid-size public university. Daily she fields questions from web editors, procurement, and leadership — today she digs through ADA.gov, section508.gov, and W3C docs manually: slow, error-prone, and risky if she paraphrases legal requirements incorrectly.

The problem: trustworthy, cited answers to compliance questions, with safe behaviour when a question is out of scope, ambiguous, or asks for legal advice — without exposing PII or fabricating policy.

Inputs, outputs & constraints

In: typed or spoken questions (EN/ES), ≤1000 chars.

Out: grounded answers with numbered citations; labeled general guidance when nothing matches; refusals; escalation hand-offs; curated checklists.

Constraints: answers only from the curated KB; no legal advice; keys server-side only; PII-redacted logs; WCAG 2.2 AA UI.

Architecture at a glance

Browser, server, and external providers — cleanly separated

One content source (the markdown KB) powers both the site pages and the assistant's retrieval; the browser never sees a provider key.

Browser — React + Vite + TS

  • Accessible UI · theme toggle
  • Visible transcript + citations
  • Keyboard + focus manager
  • Text fallback (voice is additive)
→

Server — Node + Fastify, TS

  • /api/chat → RAG + grounding
  • /api/stt → speech-to-text
  • /api/tts → streaming voice
  • Key custody · rate-limit · audit
→

External

  • OpenAI (STT, LLM)
  • ElevenLabs (TTS)
  • KB (markdown + index)
  • Keys never reach the browser. The front end talks only to this server; the server holds the keys.
  • Voice is additive. Every task is completable with keyboard + text alone.
  • Grounded + cited. Retrieval-first; a labeled model fallback only when nothing matches; off-topic is refused.
  • Provider-swappable. STT / LLM / TTS sit behind interfaces — swap via .env, no code change.
  • One content source. The same /kb/*.md files power the site pages and the bot's retrieval.

Capstone additions

Agent, tracing, and evaluation — on top of the base product

The app is an AI agent per the capstone spec (Track B, framework-free). Seven capabilities, each independently demonstrable.

Grounded RAG + citations

Retrieval-first pipeline; refusals never touch the LLM, eliminating the fabrication surface for out-of-scope input.

packages/core/retriever.js · pipeline.js · citation.js

Tool calling + safeguards

kb_search, wcag_checklist, escalate_to_human — declared schemas, loop-guarded, with failed-call fallback.

server/src/agent/tools.ts

Memory & adaptation

Session-scoped short-term memory with TTL/reset rules; thumbs-down feedback flips behaviour flags (concise / grounded-only) for the rest of the session.

server/src/lib/session.ts

Per-user, per-session tracing

Every turn produces a trace with spans (plan → tool → retrieval → llm), persisted to traces.jsonl. Anonymous identity only.

server/src/lib/tracing.ts

LLM-as-a-judge evaluation

A separate judge model scores every answer 1–5 on groundedness, citation correctness, safety, and helpfulness across a shared 16-case test set.

server/src/eval/

Safety & PII redaction

Refusal, labeled uncertainty, human escalation, and central redaction of emails/phone numbers before anything is stored — one-click safety probes in the trainer console.

/eval → Safety probes
Engineering justification

Track B: framework-free, justified

No LangChain / CrewAI / Flowise. Equivalent capabilities are implemented explicitly, keeping correctness-critical logic unit-testable offline and auditable.

Framework capabilityAccess508 equivalent
Chains / pipelinespackages/core/pipeline.js — pure, unit-tested plan: retrieve → ground → cite → prompt
Retrievers / vector storespackages/core/retriever.js (TF-IDF) + server/src/rag/embeddingRetriever.ts (OpenAI embeddings), one interface

Agent executorserver/src/agent/agent.ts — plan → tool → retrieval → LLM, every step traced
Memoryserver/src/lib/session.ts — session-scoped, TTL + reset rules
Callbacks / tracingserver/src/lib/tracing.ts — per-turn traces with spans, JSONL persistence
Evaluationserver/src/eval/ — LLM-as-judge harness + versioned prompt variants
Prompt templatespackages/core/prompt.js + server/src/eval/promptVariants.ts (v1 / v2 / v3)

Why not a framework? The product's differentiator is verifiable safety (grounding gate, citation contract, refusal path) and WCAG-grade explainability. A ~800-line explicit core is fully covered by offline unit tests (npm run test:core) — smaller than the framework glue it would replace, with no black-box behaviour to explain to auditors.

Root-cause analysis — found → fixed

PII in audit logs (found in review)
Root cause: the original auditAnswer stored a raw queryPreview; logging predated the tracing design.
Fix: central redaction (redact()) applied to every stored question/answer preview in traces — verified by the PII probe.
Baseline paraphrase misses (Phase 2 evidence)
Root cause: keyword overlap has no semantics (baseline/runs.log).
Fix: TF-IDF retrieval + optional embeddings + grounding gate in the production pipeline; judged grounded cases confirm.
Incorrect tool selection ("checklist for underwater basket weaving")
Root cause: the keyword router over-triggers on "checklist".
Fix (safeguard, not prevention): the failed tool call is traced and falls back to kb_search, capped by TOOL_MAX_ITERATIONS — detect-and-recover rather than silent failure.
Accessibility commitments

WCAG 2.2 AA, by construction

The product's own UI must be accessible to the people it serves — including users of assistive technology.

  • Semantic landmarks, skip link, visible focus, logical focus order, focus not obscured.
  • Transcript-first: every spoken turn is shown as text; voice is never required.
  • Real <button> mic toggle with matching visible + accessible name and aria-pressed.
  • Polite live region for state; role="log" transcript; citations as real links.
  • APG modal dialog: focus moves in, is trapped, Esc closes, focus returns to the launcher.
  • Dark / Light / System theme (System default, no flash-of-wrong-theme); prefers-reduced-motion honored.
  • Contrast verified: text ≥ 4.5:1, UI/non-text ≥ 3:1, on both light and dark.
  • High-contrast mode available (AAA 7:1+ ratios, heavy borders, no transparency).
